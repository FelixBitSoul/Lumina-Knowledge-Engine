# Compliance & Industry Standards

## 1. Project Guidelines

### 1.1 Follow Existing Conventions
- **Adhere to Project Conventions**: Follow the project's existing coding conventions.
- **Consistent Style**: Maintain consistent code style throughout the project.
- **Follow Documentation Standards**: Follow the project's documentation standards.
- **Respect Existing Architecture**: Respect the project's existing architecture and design decisions.

### 1.2 Code Review Checklist
- **Use a Checklist**: Use a checklist for code reviews to ensure compliance.
- **Security Review**: Include security review in code reviews.
- **Performance Review**: Include performance review in code reviews.
- **Code Quality Review**: Include code quality review in code reviews.
- **Documentation Review**: Include documentation review in code reviews.

### 1.3 Continuous Integration
- **Use CI/CD**: Use CI/CD to enforce code quality standards.
- **Automated Testing**: Run automated tests in CI/CD.
- **Code Quality Checks**: Run code quality checks in CI/CD.
- **Security Scanning**: Run security scanning in CI/CD.
- **Build Validation**: Validate builds in CI/CD.

## 2. Industry Standards

### 2.1 Follow Best Practices
- **Adhere to Industry Best Practices**: Adhere to industry best practices for the technology stack.
- **Follow Language Standards**: Follow language-specific standards and recommendations.
- **Use Recommended Tools**: Use recommended tools and libraries.
- **Stay Updated**: Keep up-to-date with security patches and best practices.
- **Follow Design Patterns**: Follow established design patterns for the technology stack.

### 2.2 Stay Updated
- **Update Dependencies**: Regularly update dependencies to fix security vulnerabilities.
- **Follow Security Advisories**: Follow security advisories for the technology stack.
- **Attend Conferences**: Attend conferences and meetups to stay updated.
- **Read Blogs**: Read blogs and articles to stay updated.
- **Join Communities**: Join online communities to stay updated.

### 2.3 Documentation Standards
- **Follow Documentation Standards**: Follow documentation standards for the technology stack.
- **Use Standard Formats**: Use standard documentation formats (e.g., Markdown, reStructuredText).
- **Maintain API Documentation**: Maintain up-to-date API documentation.
- **Document Architecture**: Document the project's architecture and design decisions.
- **Document Deployment**: Document deployment procedures and requirements.

## 3. Regulatory Compliance

### 3.1 Data Protection
- **GDPR Compliance**: Ensure compliance with GDPR if handling EU citizen data.
- **CCPA Compliance**: Ensure compliance with CCPA if handling California resident data.
- **Data Minimization**: Collect only the data necessary for the application.
- **Data Retention**: Implement appropriate data retention policies.
- **Data Breach Response**: Have a data breach response plan.

### 3.2 Industry-Specific Compliance
- **Healthcare (HIPAA)**: Ensure compliance with HIPAA if handling healthcare data.
- **Financial (PCI DSS)**: Ensure compliance with PCI DSS if handling payment card data.
- **Education (FERPA)**: Ensure compliance with FERPA if handling student data.
- **Government (FedRAMP)**: Ensure compliance with FedRAMP if working with government data.
- **Financial (SOC 2)**: Ensure compliance with SOC 2 if providing SaaS services.

- **Ignoring Compliance Requirements**: Ignoring regulatory compliance requirements.
- **Non-Compliant Data Handling**: Handling data in a non-compliant manner.
- **Inadequate Documentation**: Inadequate documentation for compliance.
- **Non-Standard Practices**: Using non-standard practices that may violate compliance.

- **Inadequate Training**: Inadequate training on compliance requirements.
