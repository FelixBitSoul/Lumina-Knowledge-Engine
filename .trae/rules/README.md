# Code Quality & Best Practices Rules

This directory contains the code quality and best practices rules for the Lumina Knowledge Engine project. The rules are organized into focused topic files for better maintainability and adherence.

## Rule Structure

- **README.md** (this file): Canonical entry point with overview and navigation
- **core.md**: Core coding principles and general best practices
- **naming.md**: Naming conventions for different languages
- **organization.md**: Module and directory organization guidelines
- **error-handling.md**: Error handling and edge case strategies
- **documentation.md**: Documentation and commenting standards
- **maintainability.md**: Code maintainability guidelines
- **performance.md**: Performance considerations and optimization
- **security.md**: Security best practices
- **compliance.md**: Industry standards and project-specific compliance

## Usage

These rules are referenced by AI coding agents and human developers to ensure consistent code quality across the project. When working on the codebase, please refer to the relevant rule files for guidance.

## Priority Order

1. **Core principles** (core.md) - foundational rules that apply to all code
2. **Language-specific rules** (naming.md) - conventions for specific programming languages
3. **Architecture rules** (organization.md) - structural guidelines
4. **Quality rules** (maintainability.md, error-handling.md) - code quality standards
5. **Specialized rules** (performance.md, security.md, compliance.md) - domain-specific guidelines

## Updating Rules

When updating rules, please:
1. Make changes in the appropriate focused file
2. Keep rules concise and actionable
3. Maintain consistency across related rules
4. Update this README if the structure changes

For more information on how to use these rules with AI coding agents, please refer to the project's AGENTS.md file.
